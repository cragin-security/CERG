# Tier 2→3 Consulting Gates
## Practice Asset — Not a CERG Corpus Document
## Purpose: Measurable criteria for upgrading a Tier 2 engagement to Tier 3

---

## Gate Model

Tier 2 engagements begin with a 90-day structural advisory. Transition to Tier 3 (compliance-to-operations) is not automatic at day 90 — it requires demonstrated operating cadence against these gate criteria. The gates prevent "readiness by calendar date" in favor of "readiness by demonstrated evidence."

---

## Gate 1: Spine Operating Cadence (Minimum — Must Pass All)

| Criterion | Evidence Required | Pass/Fail |
|---|---|---|
| Risk register has been reviewed at least 3 times with entries updated, closed, and added between cycles | Review minutes from 3 consecutive review cycles showing date, participants, changes made | ☐ |
| At least 1 risk acceptance has been formally documented and approved per RMF-001 §9.7 | Risk acceptance memo with named approver matching the authority table | ☐ |
| Architecture review procedure has processed at least 2 real projects (not test projects) | Intake forms with disposition (Approved / Conditional / Rejected), dated | ☐ |
| Exposure management has produced at least 1 SLA report showing compliance or variance | VM-001 or VM-002 dashboard snapshot dated within the last 30 days | ☐ |
| Standards gap analysis is complete; each deferred standard has a risk register entry with owner and target date | Gap analysis memo; risk register entries for deferred standards | ☐ |

**Gate 1 outcome:** All 5 pass → proceed to Gate 2. Any fail → extend Tier 2 until the criterion is met, or document as accepted risk with improvement register entry.

---

## Gate 2: Regulatory Readiness (Conditional — Required Only if Regulator in Scope)

| Criterion | Evidence Required | Pass/Fail |
|---|---|---|
| Regulatory overlay package has been selected and the CDE/relevant-system register defined | Package selection memo; system register with scope rationale | ☐ |
| Existing evidence has been mapped to regulatory requirements per the package §5 evidence reuse table | Evidence mapping worksheet; gap list identified | ☐ |
| Client has confirmed regulatory applicability with qualified counsel or compliance leadership | Signed applicability memo from client's legal/compliance team | ☐ |

**Gate 2 outcome:** All applicable criteria pass → proceed to Tier 3 scope. If regulatory packages were out of scope for the Tier 2 engagement, Gate 2 is automatically satisfied and Tier 3 is a separate engagement.

---

## Gate 3: Client Operational Readiness

| Criterion | Evidence Required | Pass/Fail |
|---|---|---|
| Client has named an accountable owner for each regulatory package (may be existing staff with new designation) | Regulatory owner list: package → named person | ☐ |
| Client has allocated calendar time for quarterly testing cycles (auditor walkthroughs, evidence collection, deficiency reviews) | Calendar holds for next 2 cycles confirmed in writing | ☐ |
| Client has designated a primary and backup for the risk register owner role | Named owners in the role consolidation map | ☐ |
| Maturity score has improved by at least 1 level since baseline (per MAT-001 scoring) or documented plateau with improvement plan | Current vs baseline maturity scorecard; improvement plan if plateaued | ☐ |

**Gate 3 outcome:** All pass → Tier 3 engagement scope is authorized. Any fail → improvement register entries with 30-day re-check.

---

## Summary: Gate Decision Matrix

| Gate 1 | Gate 2 | Gate 3 | Decision |
|---|---|---|---|
| Pass | Pass (or N/A) | Pass | Proceed to Tier 3 |
| Pass | Pass (or N/A) | Fail | 30-day remediation plan; re-check Gate 3 |
| Pass | Fail | — | Pending regulatory applicability confirmation |
| Fail | — | — | Extend Tier 2; root cause analysis |

---

## Consulting Response to Gate Failure

| Failure Pattern | Consulting Response | Fee Implication |
|---|---|---|
| Evidence gap (Gate 1) | Identify root cause: is the control not implemented, or is evidence not collected? Remediation coaching is within scope | In scope (included in original Tier 2 fee) |
| Ownership gap (Gate 3) | Facilitate executive alignment conversation; document in improvement register with escalation path | In scope for up to 2 hours; beyond that → advisory day rate |
| Regulatory readiness (Gate 2) | Provide applicability guidance; recommend external counsel engagement | Advisory day rate if outside original regulatory package selection memo |
| Maturity plateau (Gate 3) | Conduct root cause analysis: is the program running but maturing slowly, or is the cadence not operating? | In scope for one analysis session; ongoing → advisory day rate |
